Minesweeper;

  • C++
  • Simple and Fast Multimedia Library (SFML)
  • Game Development
  • Object-Oriented Programming
  • File I/O

    • Minesweeper is the second game I recreated. This was an independent project and my favorite I have done so far because it was very rewarding to see how much I had improved since coding my first game.

    Starting the project was initially very daunting because there were a lot of components to the project. There were three main windows I needed to implement: the welcome window, the game window, and the leaderboard window.

    I started working on the welcome window. Starting with implementing the welcome screen helped ease me into using the SFML library. This made rendering the game window a lot easier. The game window was certainly the most complex component of the entire project. At this step in the project, I had to implement both the game mechanics and GUI. I had to make very important decisions in this step, which included how I was going to make the individual tiles and how I was going to store them. I utilized an object-oriented programming apporoach and made a class for the tiles and stored them inside of a vector. These decisions made it easy to set attributes for each tile and made it easy to access. The last screen I needed to implement was the leaderbord window. This part utilized file I/O to read and store time scores of players.

    The biggest challenge of this project was implementing all the game mechanics. There were a lot of details of the game that needed to be implemented like taking in user input to reveal tiles or place a flag, recurssive tile revealing, and a timer. All of the details were very overwhelming at one point. To ease some of the frustration, I organized a list of tasks and tackled them one at a time. When it came to debugging, I kept a list of bugs that I found when testing my game, which made it a lot easier to keep track of and fix later.

    Looking back at the first game I recreated, which was Sudoku, I saw a large improvement in my skills. Using SFML was just a matter of learning the syntax. The concepts used were very similar to the concepts of using the PyGame library from Sudoku. Recreating Minesweeper really strengthened my game development skills and overall programming skills.
